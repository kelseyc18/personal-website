Learning about diversity, equity, and inclusion (DEI) ü§ù

I am not a DEI expert, but I've been trying to learn as much as I can about making tech more inclusive. These are some things I've worked on at my current company, Coda.

  • After being inspired by Non-Binary People's Day, I pitched the idea to add an optional pronoun field to Coda Maker Profiles. We shipped the feature soon after!
  • After learning more about the experiences of trans and non-binary people, I led discussions on how to make our women's group more inclusive to other people with marginalized genders. We also updated our mission statement as a commitment to prioritizing intersectionality since gender is only one aspect of people's identities. Read more about Gender Equity at Coda.
  • During our last company hackathon, my team learned more about website accessibility and created an accessibility audit template for unofficial audits. I also came up with some product design and engineering guidelines so that we can chip away at our accessibility debt.
  • I'm keeping a running list of DEI articles that I've found helpful. Let me know if you have any suggestions!

Why is this website so ugly? üö®

I was curious if I could make a decent website without using any custom styling. I also took this opportunity to learn how ARIA landmarks make websites more compatible with screen readers. Turns out, it's quite simple! If you are using macOS, I recommend trying to navigate this webpage using the built-in VoiceOver software. Windows users can use the built-in Narrator.

Here are two tools you can use to test the accessibility of webpages: WAVE Web Accessibility Evaluation Tool and Firefox Accessibility Inspector.

Just to clarify, websites that are accessible can still look beautiful! This one just happens to be ugly.
